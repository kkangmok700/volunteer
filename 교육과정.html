CHAPTER 1

프로젝트 소개 & 전체 구성도

무엇을 만드는지, 어떤 기술을 쓰는지 전체 그림을 먼저 파악합니다.

🎯 우리가 만들 것

삼육대학교 사회봉사단 관리 시스템은 봉사단 참여 신청부터 관리자 통계까지 모든 과정을 웹으로 처리하는 시스템입니다.

  • 신청서 페이지 — 학생들이 봉사단에 참여 신청
  • 대시보드 — 봉사단 현황, 일정, 통계 한눈에 확인
  • 관리자 페이지 — 신청 현황 관리, 데이터 조회, 차트 시각화

🗺️ 전체 아키텍처 구성도

우리 시스템은 3개의 큰 덩어리로 이루어져 있습니다:

💻 사용자 브라우저 HTML / CSS / JS
↔
🌐 GitHub Pages 정적 호스팅
↔
🗄️ Supabase 데이터베이스 + API
💡 쉽게 이해하기

GitHub Pages는 "파일 보관함"이고, Supabase는 "데이터 창고"입니다. 사용자가 웹페이지를 열면 GitHub Pages에서 화면(HTML)을 가져오고, 데이터가 필요할 때 Supabase에 물어봅니다.

🛠️ 사용 기술 스택

기술역할비유
HTML웹페이지 구조 (뼈대)건물의 골조
CSS디자인 & 스타일링건물의 인테리어
JavaScript동적 기능 (데이터 연동)건물의 전기/설비
Supabase데이터베이스 + API 서버창고 & 배달 서비스
GitHub Pages웹사이트 호스팅 (무료)건물 부지 (토지)
GitHub Actions자동 배포 (CI/CD)자동 이사 서비스

📡 웹서버 vs DB서버, 뭐가 다를까?

🌐 웹서버 (GitHub Pages)

HTML, CSS, JS 같은 파일을 전달하는 역할입니다. 식당에서 메뉴판을 건네주는 것과 같습니다. 우리 프로젝트에서는 GitHub Pages가 이 역할을 합니다.

🗄️ DB서버 (Supabase)

신청 데이터, 회원 정보 등 데이터를 저장/조회하는 역할입니다. 식당에서 주문을 기록하고 재고를 관리하는 주방 시스템 같습니다.

❗ 핵심 포인트

보통 웹 서비스는 웹서버 + DB서버 모두 유료로 운영해야 하지만, 우리는 GitHub Pages(무료) + Supabase(무료 플랜)을 사용해서 비용 0원으로 운영합니다!

개발 환경 준비

코딩을 시작하기 전에 필요한 도구들을 설치합니다.

📥 1. Git 설치 & 기본 명령어

Git은 코드의 변경 이력을 관리하는 도구입니다. "코드 타임머신"이라고 생각하면 됩니다.

  1. git-scm.com에서 Git을 다운로드하여 설치합니다.
  2. 설치 후 터미널(명령 프롬프트)을 열어 git --version으로 확인합니다.
Terminal # 자주 쓰는 Git 명령어 4가지 git init # 새 프로젝트에서 Git 시작 (처음 1번만) git add . # 변경된 파일을 "준비 상태"로 올림 git commit -m "메시지" # 변경사항을 저장 (스냅샷 찍기) git push # GitHub 서버에 업로드
💡 비유로 이해하기

git add = 택배 상자에 물건 넣기, git commit = 상자 포장하기, git push = 택배 보내기

💻 2. GitHub 계정 & 레포지토리

  1. github.com에서 무료 계정을 만듭니다.
  2. 오른쪽 상단 "+" 버튼 → "New repository" 클릭
  3. Repository name에 volunteer 입력
  4. Public 선택 (GitHub Pages 무료 사용을 위해 필수!)
  5. "Create repository" 클릭
Terminal # 로컬 프로젝트와 GitHub 연결하기 git remote add origin https://github.com/사용자명/volunteer.git git push -u origin main

📝 3. 에디터 설치

VS Code(Visual Studio Code)를 추천합니다. 무료이고, 코드 자동완성과 미리보기 기능이 있습니다.

  1. code.visualstudio.com에서 다운로드
  2. 설치 후 프로젝트 폴더를 열기 (File → Open Folder)
  3. 확장 프로그램 "Live Server" 설치하면 실시간 미리보기 가능

📂 4. 프로젝트 폴더 구조

CHAPTER 3

Supabase 데이터베이스 설정

데이터를 저장할 Supabase를 설정하고, 테이블을 만듭니다.

🚀 1. Supabase 가입 & 프로젝트 생성

  1. supabase.com에 접속하여 GitHub 계정으로 가입
  2. "New Project" 클릭
  3. 프로젝트 이름: shu-volunteer
  4. Database Password: 강력한 비밀번호 설정 (꼭 메모!)
  5. Region: Northeast Asia (ap-northeast-1) 선택 (한국에서 가장 빠름)
💡 무료 플랜으로 충분!

Supabase 무료 플랜은 500MB 데이터베이스, 무제한 API 호출을 제공합니다. 우리 프로젝트 규모에 충분합니다.

📊 2. 테이블 설계

우리 프로젝트에서 사용하는 주요 테이블입니다:

테이블명용도주요 컬럼
participants 봉사단원 정보 id, name, student_id, phone, department
applications 참여 신청 데이터 id, name, student_id, status, created_at
budget 예산 관리 id, category, amount, description, date

📝 3. SQL로 테이블 만들기

Supabase 대시보드 → SQL Editor에서 아래 SQL을 실행합니다:

SQL -- 신청 테이블 생성 CREATE TABLE applications ( id BIGSERIAL PRIMARY KEY, name TEXT NOT NULL, student_id TEXT NOT NULL, department TEXT, phone TEXT, motivation TEXT, status TEXT DEFAULT 'pending', created_at TIMESTAMPTZ DEFAULT NOW() ); -- 예산 테이블 생성 CREATE TABLE budget ( id BIGSERIAL PRIMARY KEY, category TEXT NOT NULL, amount INTEGER NOT NULL, description TEXT, date DATE DEFAULT CURRENT_DATE );

🔑 4. API 키 확인하기

Supabase와 통신하려면 2가지 정보가 필요합니다:

  1. Supabase 대시보드 → Settings → API 메뉴 이동
  2. Project URL: https://xxxx.supabase.co 형태
  3. anon (public) key: eyJhbGci...로 시작하는 긴 문자열
JavaScript // HTML 파일에서 이렇게 사용합니다 const SUPABASE_URL = 'https://xxxx.supabase.co'; const SUPABASE_KEY = 'eyJhbGci...'; // anon key
⚠️ 주의!

anon key는 공개되어도 괜찮은 키입니다(브라우저에서 사용). 하지만 service_role key는 절대 공개하면 안 됩니다! 이 키는 모든 데이터에 접근할 수 있습니다.

🔒 5. RLS (Row Level Security)

RLS는 "누가 어떤 데이터에 접근할 수 있는지"를 제어하는 보안 기능입니다.

RLS OFF (비활성)

누구나 모든 데이터를 읽고, 쓰고, 삭제할 수 있음. 개발 초기 단계에서 편리하지만 보안에 취약합니다.

RLS ON (활성)

정책(Policy)에 따라 접근이 제한됨. 예: "INSERT만 허용하고 DELETE는 차단" 같은 규칙을 설정할 수 있습니다.

SQL -- 신청 테이블: 누구나 신청(INSERT) 가능, 조회(SELECT)도 허용 ALTER TABLE applications ENABLE ROW LEVEL SECURITY; CREATE POLICY "누구나 신청 가능" ON applications FOR INSERT TO anon WITH CHECK (true); CREATE POLICY "누구나 조회 가능" ON applications FOR SELECT TO anon USING (true);
CHAPTER 4

신청서 페이지 만들기

학생들이 봉사단에 참여 신청하는 폼 페이지를 만듭니다.

📄 1. HTML 폼 구조

신청서의 기본 구조는 <form> 태그 안에 입력 필드들을 배치합니다:

HTML <form id="applicationForm"> <!-- 이름 입력 --> <div class="form-group"> <label>이름</label> <input type="text" id="name" required> </div> <!-- 학번 입력 --> <div class="form-group"> <label>학번</label> <input type="text" id="studentId" required> </div> <!-- 지원동기 --> <div class="form-group"> <label>지원동기</label> <textarea id="motivation"></textarea> </div> <button type="submit">신청하기</button> </form>

🎨 2. CSS 스타일링 (반응형)

입력 폼을 깔끔하게 꾸미는 CSS 핵심 속성들:

CSS /* 입력 필드 공통 스타일 */ input, textarea, select { width: 100%; padding: 12px 16px; border: 2px solid rgba(255,255,255,0.1); border-radius: 10px; background: rgba(255,255,255,0.05); color: #fff; font-size: 14px; transition: border-color 0.3s; } /* 포커스 시 테두리 색상 변경 */ input:focus { border-color: #e94560; outline: none; } /* 반응형: 모바일에서 폰트 크기 조정 */ @media (max-width: 480px) { input, textarea { font-size: 16px; } }
💡 모바일 팁

iOS Safari에서 font-size가 16px 미만이면 입력 시 화면이 자동 확대됩니다. 모바일을 고려한다면 최소 16px로 설정하세요.

⚡ 3. JavaScript로 Supabase 연동

폼을 제출하면 Supabase에 데이터를 저장하는 코드입니다:

JavaScript // Supabase 클라이언트 생성 (CDN 방식) const { createClient } = supabase; const db = createClient(SUPABASE_URL, SUPABASE_KEY); // 폼 제출 이벤트 document.getElementById('applicationForm') .addEventListener('submit', async (e) => { e.preventDefault(); // 입력값 가져오기 const formData = { name: document.getElementById('name').value, student_id: document.getElementById('studentId').value, motivation: document.getElementById('motivation').value }; // Supabase에 데이터 INSERT const { data, error } = await db .from('applications') .insert([formData]); if (error) { alert('오류가 발생했습니다: ' + error.message); } else { alert('신청이 완료되었습니다!'); } });

✅ 4. 폼 유효성 검사

사용자가 잘못된 데이터를 입력하는 것을 방지합니다:

JavaScript function validateForm(data) { // 이름: 2글자 이상 if (data.name.length < 2) { alert('이름을 2글자 이상 입력해주세요.'); return false; } // 학번: 숫자 형식 확인 if (!/^\d{8,}$/.test(data.student_id)) { alert('올바른 학번을 입력해주세요.'); return false; } return true; }
CHAPTER 5

관리자 페이지 만들기

신청 데이터를 조회하고 관리하는 관리자 전용 페이지를 만듭니다.

🔒 1. 로그인 기능 (localStorage)

간단한 비밀번호 기반 로그인을 구현합니다. (서버 인증 대신 클라이언트 방식)

JavaScript // 관리자 비밀번호 확인 function adminLogin() { const password = prompt('관리자 비밀번호를 입력하세요:'); if (password === '설정한비밀번호') { // 로그인 상태를 브라우저에 저장 localStorage.setItem('adminAuth', 'true'); loadAdminPage(); } else { alert('비밀번호가 틀렸습니다.'); } } // 페이지 로드 시 로그인 상태 확인 if (localStorage.getItem('adminAuth') === 'true') { loadAdminPage(); } else { showLoginScreen(); }
⚠️ 보안 참고

localStorage 기반 인증은 학습/데모 목적입니다. 실제 서비스에서는 Supabase Auth 또는 JWT 토큰 기반 인증을 사용해야 합니다.

📋 2. 데이터 조회/수정/삭제

Supabase에서 데이터를 가져오고 관리하는 기본 패턴입니다:

JavaScript // SELECT: 전체 신청 목록 조회 const { data } = await db .from('applications') .select('*') .order('created_at', { ascending: false }); // UPDATE: 신청 상태 변경 (pending → approved) await db .from('applications') .update({ status: 'approved' }) .eq('id', targetId); // DELETE: 신청 삭제 await db .from('applications') .delete() .eq('id', targetId);

📊 3. 차트 시각화 (Chart.js)

데이터를 그래프로 시각화하려면 Chart.js 라이브러리를 사용합니다:

HTML <!-- Chart.js CDN 불러오기 --> <script src="https://cdn.jsdelivr.net/npm/chart.js"></script> <!-- 차트를 그릴 캔버스 --> <canvas id="statusChart"></canvas>
JavaScript // 도넛 차트 생성 예시 new Chart(document.getElementById('statusChart'), { type: 'doughnut', data: { labels: ['승인', '대기', '거절'], datasets: [{ data: [approvedCount, pendingCount, rejectedCount], backgroundColor: ['#00d2ff', '#ffcb6b', '#e94560'] }] } });

📱 4. QR코드 생성

신청서 페이지 링크를 QR코드로 만들어 학생들에게 배포할 수 있습니다:

JavaScript // QRCode.js 라이브러리 사용 const qr = new QRCode(document.getElementById('qrcode'), { text: 'https://kkangmok700.github.io/volunteer/사회봉사단_신청서.html', width: 200, height: 200 });
⚠️ 한글 URL 주의!

URL에 한글이 포함되면 일부 QR 리더가 인식하지 못할 수 있습니다. encodeURI() 함수로 인코딩하거나, 영문 파일명을 사용하는 것이 안전합니다.

JavaScript // 한글 URL 인코딩 처리 const url = encodeURI( 'https://kkangmok700.github.io/volunteer/사회봉사단_신청서.html' ); // 결과: https://kkangmok700.github.io/volunteer/%EC%82%AC%ED%9A%8C...
CHAPTER 6

GitHub Pages 자동 배포

코드를 push하면 자동으로 웹사이트에 반영되는 CI/CD를 설정합니다.

⚙️ 1. GitHub Actions 워크플로우

.github/workflows/deploy.yml 파일을 만들어 자동 배포를 설정합니다:

YAML — deploy.yml name: Deploy to GitHub Pages on: push: branches: [ main ] permissions: contents: read pages: write id-token: write jobs: deploy: runs-on: ubuntu-latest environment: name: github-pages steps: - uses: actions/checkout@v4 - uses: actions/configure-pages@v4 - uses: actions/upload-pages-artifact@v3 with: path: '.' - uses: actions/deploy-pages@v4
💡 작동 원리

main 브랜치에 push하면 → GitHub Actions가 자동 실행 → 파일을 GitHub Pages에 업로드 → 웹사이트에 반영됩니다. 보통 1~2분이면 완료!

🔄 2. 배포 흐름 이해하기

💻 코드 수정 로컬 에디터
→
📦 git push GitHub에 업로드
→
⚙️ Actions 실행 자동 빌드 & 배포
→
🌍 웹사이트 반영 github.io

🔧 3. Pages 설정 활성화

  1. GitHub 레포지토리 → Settings 탭 클릭
  2. 왼쪽 메뉴에서 Pages 클릭
  3. Source를 "GitHub Actions"로 선택
  4. 저장 후, 다음 push부터 자동 배포가 시작됩니다
❗ Source 설정 주의

"Deploy from a branch"가 아닌 "GitHub Actions"를 선택해야 합니다. 이전 방식(branch)은 제한이 있어서 Actions 방식을 권장합니다.

🔓 4. Private vs Public 레포지토리

Public (공개)

GitHub Pages 무료 사용 가능. 누구나 코드를 볼 수 있지만, Supabase 키는 어차피 공개용(anon key)이라 문제없습니다.

Private (비공개)

코드를 숨기고 싶을 때. 하지만 GitHub Pages를 쓰려면 유료 플랜(Pro)이 필요합니다. 월 $4 정도.

💡 우리의 선택

비용 0원 운영을 위해 Public 레포를 사용합니다. 민감한 정보(비밀번호 등)는 코드에 직접 넣지 않고 Supabase RLS로 보호합니다.

🔍 5. 배포 확인 & 트러블슈팅

  1. Actions 탭에서 초록색 체크(✅) 확인 — 배포 성공
  2. 빨간색 X(❌)면 로그를 클릭해서 에러 메시지 확인
  3. 배포 URL: https://사용자명.github.io/volunteer/
⚠️ 흔한 배포 오류

404 에러: 파일명의 대소문자를 확인하세요. GitHub Pages는 대소문자를 구분합니다. Index.html과 index.html은 다른 파일입니다!

실전 트러블슈팅

실제 개발 과정에서 겪은 문제들과 해결 방법을 공유합니다.

🚨 실제 겪은 문제들

API 키를 복사할 때 잘못된 형식의 키를 사용함 (Project URL이 아닌 다른 값을 넣음)
✅ 해결 방법

Settings → API 메뉴에서 Project URL과 anon key를 정확히 복사합니다. anon key는 eyJhbGciOiJIUzI1NiIs...처럼 JWT 형식이어야 합니다. "eyJ"로 시작하지 않으면 잘못된 키입니다.

🔀 문제 2: QR코드 한글 인코딩
URL에 한글 파일명이 포함되어 일부 QR 리더에서 깨짐
✅ 해결 방법

encodeURI() 함수로 URL을 인코딩하거나, QR코드 생성 라이브러리 옵션에서 UTF-8 인코딩을 명시합니다. 가장 확실한 방법은 영문 파일명을 사용하는 것입니다.

💰 문제 3: GitHub Pages 유료 오해
Private 레포에서 Pages를 활성화하려니 유료 결제 안내가 나옴
✅ 해결 방법

레포지토리를 Public으로 변경하면 GitHub Pages를 무료로 사용할 수 있습니다. Settings → 맨 아래 "Change repository visibility"에서 변경합니다.

🔎 디버깅 팁: 브라우저 개발자 도구 (F12)

웹 개발에서 가장 중요한 도구는 브라우저 개발자 도구입니다. F12 키로 열 수 있습니다.

탭용도자주 쓰는 상황
Console 에러 메시지 확인, JS 실행 "왜 버튼이 안 눌리지?" → 콘솔에 에러 메시지 확인
Network API 요청/응답 확인 "데이터가 왜 안 나오지?" → Supabase 요청 상태 확인
Elements HTML/CSS 실시간 수정 "이 버튼 색상을 바꿔보자" → CSS 실시간 테스트
Application localStorage 등 확인 "로그인 상태가 저장됐나?" → localStorage 값 확인
Console 활용법 // 1. 데이터 확인할 때 console.log('받은 데이터:', data); // 2. 에러 추적할 때 try { // 에러가 날 수 있는 코드 } catch (error) { console.error('에러 발생:', error); } // 3. 네트워크 요청 확인 (Network 탭) // - Status 200: 성공 // - Status 401: 인증 오류 (API 키 확인) // - Status 404: URL 오류 (주소 확인)
💡 디버깅 3단계 법칙

1단계: F12 → Console에서 빨간 에러 메시지 확인
2단계: Network 탭에서 API 요청이 제대로 되는지 확인
3단계: 에러 메시지를 구글에 검색 (복사 → 붙여넣기)
